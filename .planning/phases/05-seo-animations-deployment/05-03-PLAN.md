Per SEOP-05, Lighthouse scores must exceed 90 on Performance, Accessibility, Best Practices, and SEO.

**1. Run Lighthouse CI audit against the production site:**
```bash
npx -y lighthouse https://mokshverma.in --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless --no-sandbox"
```

If Lighthouse CLI has issues running headless, try:
```bash
npx -y lighthouse https://mokshverma.in --output=html --output-path=./lighthouse-report.html --chrome-flags="--headless=new --no-sandbox"
```

**2. Parse and check scores:**
```bash
node -e "
const r = require('./lighthouse-report.json');
const cats = r.categories;
console.log('Performance:', Math.round(cats.performance.score * 100));
console.log('Accessibility:', Math.round(cats.accessibility.score * 100));
console.log('Best Practices:', Math.round(cats['best-practices'].score * 100));
console.log('SEO:', Math.round(cats.seo.score * 100));
const allAbove90 = Object.values(cats).every(c => c.score >= 0.9);
console.log('All > 90:', allAbove90);
"
```

**3. If any score is below 90, apply targeted fixes:**

Common fixes for each category:
- **Performance < 90**: Check if images need `next/image` optimization, verify no render-blocking resources, check for large JS bundles. The build is already lean (~102kB shared + ~30kB motion).
- **Accessibility < 90**: Verify all images have alt text, all interactive elements are keyboard accessible, color contrast meets WCAG AA. The existing code already uses semantic HTML and ARIA labels.
- **Best Practices < 90**: Check for console errors, ensure HTTPS, verify no deprecated APIs.
- **SEO < 90**: Verify meta tags are present (should be covered by Plan 01), check for missing alt attributes, verify canonical URL.

**4. If the site is not yet deployed** (Task 2 checkpoint not completed), run Lighthouse against the local dev server instead:
```bash
npx next build && npx next start &
sleep 3
npx -y lighthouse http://localhost:3000 --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless --no-sandbox"
kill %1
```

**5. Clean up temporary files:**
```bash
rm -f lighthouse-report.json lighthouse-report.html
```

Note: Lighthouse scores can vary between runs. Run 2-3 times and use the median if scores are borderline. The existing codebase is already well-optimized (small bundle, semantic HTML, optimized fonts), so scores >90 are expected without major changes.
